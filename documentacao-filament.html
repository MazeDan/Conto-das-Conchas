Painel Administrativo: Criação de Poemas

Documentação técnica da estrutura do formulário em FilamentPHP v3, detalhando o layout em Grid e o ciclo de vida dos dados.

1. Visão Geral da Arquitetura do Formulário

O formulário de criação de poemas foi projetado para espelhar a experiência de um CMS moderno. Em vez de empilhar todos os campos verticalmente, utilizamos o componente Grid do Filament para dividir a tela:

  • Coluna Principal (2/3 da tela): Focada puramente na escrita e identidade da obra (Título, Slug e Versos).
  • Coluna Lateral (1/3 da tela): Focada nas configurações de mídia e publicação (Imagem, Status e Data).

2. O Código do Schema (PostForm)

Abaixo está a estrutura otimizada da classe que define os campos do formulário:

public static function form(Form $form): Form
{
    return $form
        ->schema([
            Grid::make(3)->schema([
                
                // --- COLUNA PRINCIPAL (Conteúdo) ---
                Section::make('Conteúdo da Obra')
                    ->schema([
                        TextInput::make('title')
                            ->required()
                            ->live(onBlur: true)
                            ->afterStateUpdated(fn ($state, callable $set) =>
                                $set('slug', Str::slug($state))
                            ),

                        TextInput::make('slug')
                            ->required()
                            ->unique(ignoreRecord: true),

                        RichEditor::make('content')
                            ->required()
                            ->toolbarButtons(['bold', 'italic', 'undo', 'redo'])
                            ->helperText('Dica: Use "Shift + Enter" para pular apenas uma linha.')
                            ->columnSpanFull(),
                    ])->columnSpan(2),

                // --- COLUNA LATERAL (Configurações) ---
                Section::make('Configurações')
                    ->schema([
                        FileUpload::make('cover_image')
                            ->image()
                            ->directory('posts')
                            ->imageEditor()
                            ->visibility('public'),

                        Toggle::make('published')
                            ->live()
                            ->default(false),

                        DateTimePicker::make('published_at')
                            ->visible(fn ($get) => $get('published')),
                    ])->columnSpan(1),
            ])
        ]);
}

Decisões Técnicas Importantes:

  • Geração Dinâmica de Slug: O uso de live(onBlur: true) no campo de título instrui o Livewire a ir ao servidor gerar o slug automaticamente assim que o usuário tira o foco do campo, sem sobrecarregar o servidor a cada tecla digitada.
  • RichEditor Minimalista: Como se trata de poesia, reduzimos os botões da barra de ferramentas (toolbarButtons) para evitar formatações que quebrem o design. A instrução do Shift + Enter é crucial para inserir tags <br> em vez de parágrafos completos <p>.
  • Campos Reativos: O DateTimePicker de publicação só aparece na tela se o Toggle de "Publicar" for ativado. Isso é feito combinando o live() no Toggle com o visible(fn ($get) => ...) na data.

3. O Ciclo de Vida: Injetando o User ID

Um erro muito comum no Filament é tentar alterar os dados antes de salvar diretamente na classe do Formulário. O local correto para intervir no ciclo de vida (como injetar o autor do post) é nas classes das Pages (Páginas).

No arquivo app/Filament/Resources/PostResource/Pages/CreatePost.php, utilizamos o Hook mutateFormDataBeforeCreate:

protected function mutateFormDataBeforeCreate(array $data): array
{
    // Pega o ID do usuário logado e atrela à criação do poema
    $data['user_id'] = auth()->id();
    
    return $data;
}

Importante: Se você quiser que o user_id também seja preservado ou manipulado durante a edição de um poema, esse mesmo Hook precisará ser replicado no arquivo EditPost.php (usando mutateFormDataBeforeSave).
