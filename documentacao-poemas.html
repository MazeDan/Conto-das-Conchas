Leitor de Poemas: Master-Detail SPA

Documentação técnica da interface responsiva construída com Laravel Blade, Tailwind CSS e Alpine.js.

1. Visão Geral e Arquitetura

Esta view implementa um padrão de interface chamado Master-Detail (Mestre-Detalhes). Em vez de carregar uma nova página a cada clique (padrão tradicional), ela se comporta como uma SPA (Single Page Application). Todos os poemas são carregados no HTML, e a interface apenas alterna qual deles está visível no momento.

  • Laravel Blade: Motor de templates que injeta os dados do backend ($posts).
  • Tailwind CSS: Framework utilitário para estilização e responsividade direto no HTML.
  • Alpine.js: Framework JavaScript minimalista que gerencia o estado da interface (mostrar/esconder poemas) sem a complexidade de criar componentes separados em Vue ou React.

2. O Cérebro da Interface: Alpine.js

A magia da interatividade acontece logo na primeira div principal que envelopa todo o layout:

<div x-data="{ activePoem: null }">
  • x-data: Cria um "estado" local na página. A variável activePoem guarda o ID do poema que o leitor quer visualizar. Ele começa como null (estado vazio, nenhum poema selecionado).

Quando o leitor clica em um botão no índice (o menu lateral):

<button @click="activePoem = {{ $post->id }}; window.scrollTo({ top: 0, behavior: 'smooth' })">
  • @click: O Alpine intercepta o clique, altera o valor de activePoem para o ID daquele post específico e, em seguida, rola a página suavemente para o topo. Todos os elementos do HTML que dependem do activePoem reagem instantaneamente a essa mudança.

3. O Esqueleto e a Responsividade (Mobile vs. Desktop)

O layout utiliza Flexbox em sua estrutura base para se adaptar a qualquer tamanho de tela sem quebrar a proporção visual.

<div class="flex flex-col md:flex-row">
  • No Celular (flex-col): Os blocos se empilham verticalmente. A visibilidade é controlada dinamicamente: se há um poema aberto, o Alpine esconde o índice. Se o poema é fechado, o índice volta a aparecer.
  • No Computador (md:flex-row): A partir do breakpoint médio (telas maiores que 768px), a tela se divide em duas colunas. O Índice ocupa um terço (md:w-1/3) e a leitura ocupa o resto (md:w-2/3).

4. Dissecando os Componentes

A. O Estilo Base (CSS Customizado)

[x-cloak] { display: none !important; }
.bg-paper { 
    background-color: #F9F8F6; 
    background-image: radial-gradient(#e5e5e5 1px, transparent 1px); 
}

x-cloak: Previne o temido "FOUC" (Flash of Unstyled Content). Essa classe garante que o HTML não vomite todos os poemas de uma vez na tela durante a fração de segundo que o Alpine.js leva para ser carregado e esconder os elementos.

B. O Índice (Menu Lateral)

<aside class="md:sticky md:top-0 md:h-screen md:overflow-y-auto scrollbar-hide">

A classe md:sticky md:top-0 faz com que o menu fique preso e acompanhe a rolagem no desktop, garantindo que a lista de poemas esteja sempre acessível independente do quão longo seja o texto lido ao lado.

C. As Imagens em Miniatura (Avatares)

<div class="shrink-0 w-10 h-10 overflow-hidden rounded-full ring-2 ring-amber-300">
    <img src="..." class="w-full h-full object-cover object-center">
</div>

A combinação de w-full h-full object-cover dentro de uma div com tamanho fixo garante que a imagem preencha a miniatura perfeitamente e sofra um corte limpo nas bordas, sem nunca ser espremida ou distorcida.

D. As Animações de Entrada

<article 
    x-cloak 
    x-show="activePoem === {{ $post->id }}" 
    x-transition:enter="transition ease-out duration-700"
    x-transition:enter-start="opacity-0 translate-y-8"
    x-transition:enter-end="opacity-100 translate-y-0"
>

As diretivas de x-transition configuram um fade-in elegante. O poema começa invisível e um pouco mais abaixo no eixo Y (translate-y-8) e desliza suavemente para a posição original enquanto se torna totalmente opaco.

5. Boas Práticas e Acessibilidade (Nível Produção)

  • Performance (loading="lazy"): Adicionado em todas as tags de imagem. Como o @foreach injeta todos os poemas no HTML ao mesmo tempo, o atributo avisa ao navegador do usuário para baixar a imagem apenas se o poema for de fato aberto, economizando muitos dados e acelerando o carregamento inicial da página.
  • Acessibilidade por Teclado: As tags <button> ganharam as diretivas focus:ring-2 focus:outline-none. Isso garante que usuários navegando pela tecla Tab saibam exatamente qual poema está selecionado.
  • Preservação de Versos (whitespace-pre-wrap): Fundamental para sites de literatura. Essa classe do Tailwind avisa ao navegador para respeitar as quebras de linha exatas (os "Enters") e recuos presentes no banco de dados, não transformando tudo num bloco de texto massivo e respeitando a métrica poética visual.
